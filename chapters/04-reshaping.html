4  Reshaping and Tidying

The previous chapters taught you to load, summarize, and combine data. This chapter addresses a subtler but equally important skill: reshaping data between wide and long formats. The same underlying data can be arranged in multiple ways, and the shape you need depends on what you want to do with it. Groupby-aggregation and pivot tables often produce wide tables that are easy to read, but visualization libraries like seaborn expect long tables with one observation per row. Being able to move fluently between these formats is essential.

We will use monthly passenger traffic data from Denver International Airport (DIA) to illustrate these operations. The COVID-19 pandemic caused dramatic disruptions to air travel, making this dataset both analytically interesting and motivating.

4.1 Setup

import numpy as np
import pandas as pd

idx = pd.IndexSlice

pd.options.display.max_columns = 100

4.2 The Data

4.2.1 Formulate your question

How did the COVID-19 shutdowns impact passenger and flight traffic at Denver International Airport?

4.2.2 Read in data

passenger_df = pd.read_csv('passengers.csv')
passenger_df.head()
passenger_df.tail()

4.2.3 Check the packaging

The data should span from 1995 (when DIA opened) through late 2021. A simple but fragile way to check this is to sort the “Month” column — but string sorting on dates gives wrong results (e.g., “April” comes before “January” alphabetically). Instead, convert to proper Timestamp objects using pd.to_datetime.

passenger_df['Month'] = pd.to_datetime(passenger_df['Month'])

passenger_df['Month'].min(), passenger_df['Month'].max()

4.2.4 Clean up the data

Make the “Month” column the index and drop the “Cargo” column, which is not relevant to our passenger analysis.

passenger_df.set_index('Month', inplace=True)
passenger_df.drop(columns=['Cargo'], inplace=True)
passenger_df.head()

4.2.5 Validate against an external source

These statistics are collected by the DIA authority. The U.S. Department of Transportation’s Bureau of Transportation Statistics also publishes historical passenger statistics by airport. Let’s compare.

transstats_df = pd.read_excel(
    'transtats_passengers.xlsx',
    sheet_name=0,
    header=1
)

# Filter out annual total rows and null rows
transstats_df = transstats_df[transstats_df['Month'] != "TOTAL"]
transstats_df.dropna(subset=['Month'], inplace=True)

transstats_df.head()

Combine the Year and Month columns into a proper datetime. This is a common data-cleaning task when dates are split across columns.

string_dates = transstats_df[['Year', 'Month']].apply(
    lambda x: '{Year}-{Month:02d}'.format(**x), axis=1
)

transstats_df['Date'] = pd.to_datetime(string_dates)
transstats_df.set_index('Date', inplace=True)

transstats_df.head()

Join them together for comparison (practice from Chapter 3).

dia_bts_merge_df = pd.merge(
    left=passenger_df,
    right=transstats_df,
    left_index=True,
    right_on='Date',
    how='right'
)

dia_bts_merge_df.tail()

Calculate the correlation between the monthly totals from each dataset. Not perfect, but more than high enough to trust.

dia_bts_merge_df[['Grand', 'TOTAL']].corr()

4.2.6 Make a plot

Visualize the monthly total passenger data. We will go into visualization in much more detail in Chapter 6 and beyond.

%matplotlib inline
import matplotlib.pyplot as plt

ax = passenger_df['Grand'].plot()

ax.axvline(pd.Timestamp('2001-09-01'), color='r', linestyle='--', linewidth=1, zorder=-1)
ax.axvline(pd.Timestamp('2020-03-01'), color='r', linestyle='--', linewidth=1, zorder=-1)

4.2.7 Try an easy solution

How much did passenger traffic decrease in March 2020 compared to March 2019?

total_march_2019 = passenger_df.loc['2019-03-01', 'Grand']
total_march_2020 = passenger_df.loc['2020-03-01', 'Grand']

"{0:.1%}".format((total_march_2020 - total_march_2019) / total_march_2019)

4.3 Stacking: Wide to Long

If you have a DataFrame and want to convert its columns into index levels, use pandas’s .stack() (docs, user guide).

Our passenger_df is currently in wide format: each airline is a column, each row is a month, and the values are passenger counts.

passenger_df.head()

Stacking converts this into a long format — a MultiIndexed Series with Month at the first level and airline at the second level.

passenger_stack_df = passenger_df.stack()

passenger_stack_df.head(10)

Turn this MultiIndexed Series into a flat DataFrame with .reset_index(), then rename the columns for clarity.

passenger_stack_df = passenger_stack_df.reset_index()

passenger_stack_df.head(2)
passenger_stack_df.rename(
    columns={
        'level_1': 'Airline',
        0: 'Passengers'
    },
    inplace=True
)

passenger_stack_df.head()

This long format — where each row is a single observation (one airline, one month, one passenger count) — is what Hadley Wickham calls tidy data. Each variable forms a column, each observation forms a row, and each type of observational unit forms a table.

TipTidy data and visualization

Seaborn and many other visualization libraries expect tidy (long) data. If a plot function asks for x, y, and hue parameters that all reference column names, you need your data in long format.

4.4 Unstacking: Long to Wide

Now we want to reverse the process — take our long DataFrame and return it to wide format. Set the columns you want as index levels using .set_index(), then call .unstack() to pivot one level into columns.

passenger_unstack_df = passenger_stack_df.set_index(['Month', 'Airline'])

passenger_unstack_df
passenger_unstack_df = passenger_unstack_df.unstack('Airline')

passenger_unstack_df.head()

This output is a little annoying because it created a MultiIndex on the columns — see the “Passengers” above the airline names? Accessing United requires passing passenger_unstack_df[('Passengers', 'United')]. You can remove this extra level with .droplevel().

passenger_unstack_df.droplevel(0, axis='columns')

The cleaner alternative is to unstack a Series rather than a DataFrame. Access the single value column as a Series first, then unstack.

passenger_stack_s = passenger_stack_df.set_index(['Month', 'Airline']).loc[:, 'Passengers']

passenger_stack_s
passenger_stack_s.unstack(1)

This produces a clean, simple wide DataFrame without the MultiIndex on columns.

4.5 Melting: Another Path from Wide to Long

The pd.melt function offers a more explicit interface for going from wide to long. You specify which columns remain as identifiers (id_vars) and which get “melted” into value rows.

First, reset the index to get “Month” as a column.

passenger_df_reset = passenger_df.reset_index()
print(passenger_df_reset.shape)
passenger_df_reset.head()

Note the shape: 321 rows × 9 columns. After melting, 8 of those 9 columns (the airlines) should become rows. So we should get approximately 321 × 8 = 2,568 rows.

321 * 8
passengers_melt_df = pd.melt(
    frame=passenger_df_reset,
    id_vars=['Month'],
    var_name='Airline',
    value_name='Count'
)

print(passengers_melt_df.shape)
passengers_melt_df.head()

You can also pass explicit value_vars to melt only specific columns.

passengers_melt_df = pd.melt(
    frame=passenger_df_reset,
    id_vars=['Month'],
    value_vars=['American', 'Delta', 'Frontier', 'Other',
                'Southwest', 'United', 'United Express', 'Grand'],
    var_name='Airline',
    value_name='Count'
)

print(passengers_melt_df.shape)
passengers_melt_df.head()
WarningChoose your tool

.stack() and .melt() both go from wide to long, but .melt() gives you more control over which columns to include and how to name the result. Use .melt() when you want to reshape a subset of columns; use .stack() when you want to convert all value columns.

4.6 Unmelting with Pivot

A melted DataFrame can be “unmelted” with the .pivot() method. Pass the column names you want for the index, columns, and values.

passengers_melt_df.pivot(index='Month', columns='Airline', values='Count').head()

You can also pivot the other way — airlines as rows, months as columns.

passengers_melt_df.pivot(index='Airline', columns='Month', values='Count')

4.7 Summary of Reshaping Operations

Here is a quick reference for how the reshaping operations relate to each other:

  • Wide → Long: .stack() or pd.melt()
  • Long → Wide: .unstack() or .pivot() or pd.pivot_table()
  • Wide → Long → Wide should return you to where you started (a good sanity check)
