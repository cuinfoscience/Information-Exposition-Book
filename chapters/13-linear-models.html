13  Linear Models

In Chapter 12 you visualized the relationships between pairs of continuous variables and described them qualitatively. This chapter takes the next step: quantifying those relationships with linear regression models. Rather than saying “education and income appear positively related,” you will be able to say “each additional percentage point of college graduates in a county is associated with a $1,200 increase in median income, and the model explains 35% of the variation.”

We will estimate regression models using three different Python libraries — scipy, scikit-learn, and statsmodels — each of which offers different strengths. We will also use seaborn’s regplot and lmplot functions to combine regression estimation with visualization, examine residuals to diagnose model quality, and introduce multiple regression.

13.1 Setup

import numpy as np

import pandas as pd
idx = pd.IndexSlice

%matplotlib inline
import matplotlib.pyplot as plt
from matplotlib import ticker

from scipy import stats
import statsmodels.api as sm

We continue using the county-level dataset from Chapter 12.

counties_df = pd.read_csv('us_counties.csv', dtype={0: str}, na_values=['-'])
counties_df.rename(columns={'Unnamed: 0': 'FIPS'}, inplace=True)
counties_df.set_index('county_fips', inplace=True)
print(counties_df.shape)
counties_df.head()

13.3 The Linear Model

A linear regression estimates the parameters of the equation:

\[y = \alpha + \beta x\]

where \(y\) is the outcome variable, \(x\) is the input variable, \(\beta\) is the slope (how much \(y\) changes for each one-unit increase in \(x\)), and \(\alpha\) is the intercept (the predicted value of \(y\) when \(x = 0\)). The parameters are estimated using a method called ordinary least squares (OLS), which finds the line that minimizes the sum of squared differences between observed and predicted values.

13.4 Estimation with scipy.stats.linregress

The simplest interface is scipy.stats.linregress, which takes two arrays and returns the slope, intercept, correlation coefficient (\(r\)), p-value, and standard error.

f, ax = plt.subplots()

sb.regplot(
    data=counties_df,
    x='Pct_Born_Same_State',
    y='Pct_HS_Grad',
    line_kws={'color': 'tab:red'},
    ax=ax
)

ax.set_xlim((0, 100))
model = stats.linregress(
    x=counties_df['Pct_Born_Same_State'].fillna(counties_df['Pct_Born_Same_State'].mean()),
    y=counties_df['Pct_HS_Grad'].fillna(counties_df['Pct_HS_Grad'].mean())
)

model

13.4.1 Interpreting the parameters

Each parameter has a substantive meaning.

The slope of approximately 0.37 means that each additional percentage point of a county’s population born in the same state is associated with a 0.37 percentage point increase in high school graduation rates.

The intercept of approximately 58 means that the model predicts a high school graduation rate of 58% for a hypothetical county where 0% of the population was born in the same state.

The r-value can be squared to get \(R^2\), the coefficient of determination — the proportion of variance in \(y\) explained by \(x\).

model.rvalue ** 2

The p-value tests the null hypothesis that the slope is zero (no relationship). A very small p-value indicates strong evidence against this null.

WarningMissing data and regression

linregress does not handle NaN values — it will return NaN for all parameters if any input contains missing data. You must handle missing data before fitting, either by dropping rows with .dropna() or imputing with .fillna(). The choice matters and should be documented.

13.4.2 Visualizing the model

Plot the data with the estimated regression line overlaid.

f, ax = plt.subplots(figsize=(6, 6))

counties_df.plot.scatter(x='Pct_Born_Same_State', y='Pct_HS_Grad', ax=ax, s=25)

_x = counties_df['Pct_Born_Same_State'].fillna(0)
_y = model.intercept + model.slope * _x
ax.plot(_x, _y, color='tab:red')
ax.set_xlim((0, 100))
ax.set_ylim((0, 60))

13.5 Estimation with scikit-learn

For more complex models, scikit-learn’s LinearRegression uses the fit/predict pattern familiar throughout the machine learning ecosystem.

from sklearn.linear_model import LinearRegression

X = counties_df['Pct_Born_Same_State'].fillna(
    counties_df['Pct_Born_Same_State'].mean()
).values.reshape(-1, 1)

y = counties_df['Pct_HS_Grad'].fillna(
    counties_df['Pct_HS_Grad'].mean()
).values.reshape(-1, 1)

sk_reg = LinearRegression().fit(X, y)

The model statistics are stored as attributes.

sk_reg.coef_
sk_reg.intercept_
sk_reg.score(X, y)

Note that scikit-learn requires the predictor to be a 2D array (hence .reshape(-1, 1)), even for simple linear regression, because the library is designed for models with multiple predictors.

13.6 Estimation with statsmodels

The statsmodels library provides the most detailed statistical output, including confidence intervals, diagnostic tests, and a familiar summary table.

_cleaned_df = counties_df.copy()
_cleaned_df['Pct_HS_Grad'] = _cleaned_df['Pct_HS_Grad'].fillna(_cleaned_df['Pct_HS_Grad'].mean())
_cleaned_df['Pct_Born_Same_State'] = _cleaned_df['Pct_Born_Same_State'].fillna(
    _cleaned_df['Pct_Born_Same_State'].mean()
)

sm_reg = smf.ols('Pct_HS_Grad ~ Pct_Born_Same_State', data=_cleaned_df).fit()

print(sm_reg.summary())

The formula interface (y ~ x) is borrowed from R and is intuitive once you get used to it. The summary table includes everything from linregress plus confidence intervals for each coefficient, F-statistics, and diagnostic measures.

13.8 Residuals

Unless you have a very simple mechanistic relationship, there is residual variance — the difference between the model’s prediction and the actual observation. Well-performing models have small residuals; poor models have large ones.

model = stats.linregress(
    x=counties_df['Pct_Family_Poverty'].fillna(counties_df['Pct_Family_Poverty'].mean()),
    y=counties_df['Pct_Unemployed'].fillna(counties_df['Pct_Unemployed'].mean())
)

_x = counties_df['Pct_Family_Poverty'].fillna(counties_df['Pct_Family_Poverty'].mean())
_y_pred = model.intercept + model.slope * _x

counties_df['residuals'] = _y_pred - counties_df['Pct_Unemployed'].fillna(
    counties_df['Pct_Unemployed'].mean()
)

13.8.1 Diagnosing model quality

A well-specified regression model should have residuals that are independent, have constant variance, are normally distributed, and have a mean of zero.

Residuals vs. predicted values — look for patterns. A random scatter is good; a funnel or curve is bad.

f, ax = plt.subplots(figsize=(6, 6))

sb.regplot(x=_y_pred, y=counties_df['residuals'],
           scatter_kws={'alpha': .5}, line_kws={'color': 'tab:red'}, ax=ax)
ax.axhline(0, c='tab:red', lw=2, ls='--')
ax.set_xlabel('Predictions')
ax.set_ylabel('Residuals')

Histogram of residuals — should be approximately normal and centered on zero.

f, ax = plt.subplots(figsize=(6, 6))

counties_df['residuals'].hist(bins=25, ax=ax)
ax.grid(None)
ax.axvline(counties_df['residuals'].mean(), c='tab:green', lw=2)
ax.axvline(0, c='tab:red', lw=2, ls='--')
ax.set_xlabel('Residual')
ax.set_ylabel('Count')

13.9 Multiple Regression

Often you want to estimate an outcome as a function of more than one input variable. statsmodels provides the most intuitive interface for multiple regression via the formula syntax.

one_X_fit = smf.ols('Pct_HS_Grad ~ Median_Income', data=counties_df.fillna(0)).fit()
print(one_X_fit.summary())
two_X_fit = smf.ols('Pct_HS_Grad ~ Pct_Born_Same_State + Median_Income',
                      data=counties_df.fillna(0)).fit()
print(two_X_fit.summary())

Notice that the coefficient for Median_Income changes when Pct_Born_Same_State is added to the model. This is because the two predictors are correlated, and the multiple regression partitions the explained variance between them. The coefficient in a multiple regression represents the effect of each variable holding the other variables constant.

TipInterpreting multiple regression

In a multiple regression, each coefficient represents the expected change in \(y\) for a one-unit increase in that predictor, holding all other predictors constant. This is different from the simple regression coefficient, which does not control for other variables.

$$y = \alpha + \beta x$$

## Multiple Regression

Often you want to estimate an outcome as a function of more than one input variable. `statsmodels` provides the most intuitive interface for multiple regression via the formula syntax.

```{python}
one_X_fit = smf.ols('Pct_HS_Grad ~ Median_Income', data=counties_df.fillna(0)).fit()
print(one_X_fit.summary())
```

```{python}
two_X_fit = smf.ols('Pct_HS_Grad ~ Pct_Born_Same_State + Median_Income',
                      data=counties_df.fillna(0)).fit()
print(two_X_fit.summary())
```

Notice that the coefficient for `Median_Income` changes when `Pct_Born_Same_State` is added to the model. This is because the two predictors are correlated, and the multiple regression partitions the explained variance between them. The coefficient in a multiple regression represents the effect of each variable *holding the other variables constant*.

::: {.callout-tip}
## Interpreting multiple regression
In a multiple regression, each coefficient represents the expected change in $y$ for a one-unit increase in that predictor, *holding all other predictors constant*. This is different from the simple regression coefficient, which does not control for other variables.
:::

## Exercise 4: Multiple regression

Estimate a model predicting `Pct_HS_Grad` from both `Pct_Born_Same_State` and `Median_Income` using `statsmodels`. Compare the coefficients to the simple regressions. How does the $R^2$ change?
